# curso-git-semana3

## Diferencia entre merge y rebase

Git merge une los cambios de una rama con otra y conserva todo el historial de commits, creando un commit de unión.

Git rebase reorganiza los commits de una rama sobre otra, dejando un historial más limpio y lineal.

La principal diferencia es que merge conserva el historial completo y rebase lo reordena.

## ¿Cuándo usar stash?

Git stash se utiliza cuando se tienen cambios sin guardar y se necesita cambiar de rama sin hacer commit.

Permite guardar temporalmente esos cambios y recuperarlos después.

## ¿Para qué sirven los tags?

Los tags sirven para marcar versiones importantes del proyecto.

Se usan para identificar versiones estables o lanzamientos, como la versión v1.0.
